TP2: Chats de WhatsApp

Consigna

Se pide implementar un programa que a partir de un archivo de chat de WhatsApp permita encontrar estadísticas y generar mensajes pseudo-aleatorios simulando la forma de escribir de los contactos del chat.

El flujo del programa deberá ser el siguiente:

  1. Pedirle al usuario la ruta de un archivo de chat.
  2. Mostrar un menú con opciones que permitan:
    1. A partir de una serie de palabras, contar las veces que cada contacto dijo cada palabra, y guardar los resultados en un .csv.
    2. A partir de un contacto, generar un mensaje pseudo-aleatorio.
    3. Salir del programa

Archivos de chat

Lo primero que debe hacer el programa es leer el archivo de un chat.

La aplicación WhatsApp permite, desde el dispositivo móvil, la opción de seleccionar cualquier chat y exportar los mensajes en un archivo de texto. Para los fines del trabajo práctico nos va a interesar leer el contenido del archivo para identificar a cada contacto y mensaje asociado.

A modo de ejemplo se adjuntan varios archivos. Como el formato es diferente para dispositivos iPhone o Android, se ofrecen dos empaquetados. Para los fines del trabajo práctico, se debe elegir uno de los dos formatos para que su programa pueda extraer la información relevante.

chats-de-android.zip chats-de-iphone.zip

Además del conjunto de archivos ejemplo ofrecidos por el enunciado, ustedes también pueden exportar sus chats para pasarlos por el programa. El procedimiento para hacerlo es:

  1. Abrir el chat individual o grupal.
  2. Abrir el menú por los tres puntos de arriba a la derecha.
  3. Seleccionar Más > Exportar chat.
  4. Exportar el chat sin multimedia.

Es posible que el procedimiento haya devuelto más de un archivo. Acá solo se debe utilizar el .txt que corresponde a los mensajes del chat.

Consideraciones:

  • Si te sale algún error relacionado a la codificación del archivo al abrir e iterarlo, intentá abrirlo con with open(ruta, 'r', encoding='utf8').
  • Tener en cuenta que el formato es complejo. Mensajes como <Multimedia omitido> idealmente deberían ignorarse, aunque esto no es obligatorio.
  • A lo largo del trabajo práctico, se considera una palabra como toda secuencia de caracteres sin espacios en el medio. Entonces para el mensaje hola! como estás? las palabras serían "hola!", "como", "estás?".
  • Palabras con tildes, con errores de ortografías, o caracteres especiales, se pueden consideran como palabras diferentes. Entonces el mensaje cómo como comoo como? tiene cuatro palabras que consideramos diferentes.
  • Palabras con el mismo contenido que difieran en sus mayúsculas o minúsculas las consideramos como iguales. Entonces el mensaje Hola hola tiene dos palabras que consideramos iguales.
  • Si un mensaje tiene más de una línea, se considera solo la primera.
  • Si en una línea del archivo no podemos extraer al contacto y el mensaje que mandó, se ignora dicha línea.
  • Se puede considerar que el nombre de contacto no tiene caracteres especiales.

Para resumir las consideraciones: el programa deberá hacer el mejor esfuerzo para leer y extraer los mensajes con sus respectivos contactos. Si no logra filtrar o seleccionar las líneas de forma acorde, puede simplemente ignorar las líneas correspondientes sin que el programa explote.

Contar palabras por contacto

Al seleccionar la opción de contar palabras por contacto, el programa deberá solicitar una serie de palabras a contar junto con un archivo destino para guardar los resultados. Ejemplo:

Ingrese las palabras para contar entre contactos: hola como estas
Ingrese el archivo destino para guardar el reporte: mis-resultados.csv
Reporte generado!

Lo cuál genera un archivo con el siguiente formato, donde el orden de las líneas no es relevante:

Don José,como,140
Don Pepito,estas,19
Don José,hola,42
Don Pepito,hola,43

Considerando un ejemplo más reducido, tomando por ejemplo los siguientes mensajes:

Don Pepito: Hola señor don josé, como se encuentra en este hermoso día?
Don Pepito: Hola hola holalá
Don José: Señor don pepito, hola yo me encuentro muy bien.

Don Pepito,como,1

Generación de mensajes

Al seleccionar la opción de generación de mensajes, se deberá indicar al usuario los posibles contactos para que ingrese uno y luego se genere un mensaje pseudo-aleatorio. Posible ejemplo de interacción:

Contactos:
        0. Don Pepito
        1. Don José
        2. Salir

Ingrese el contacto para generar el mensaje: 0

"""Don Pepito: y tengo queso rallado así se queja"""

Ingrese el contacto para generar el mensaje: 1

"""Don José: saliendo! nos vemos bien"""

Para lograr la generación de mensajes pseudo-aleatorios, utilizaremos Cadenas de Markov.

Cadenas de Markov

Wikipedia define una Cadena de Markov de esta manera:

En la teoría de la probabilidad, se conoce como cadena de Markov o modelo de Markov a un tipo especial de proceso estocástico discreto en el que la probabilidad de que ocurra un evento depende solamente del evento inmediatamente anterior. Esta característica de falta de memoria recibe el nombre de propiedad de Markov.

Aplicación a la generación de textos

Si consideramos como evento a la elección de una palabra, podemos modelar nuestro problema con Cadenas de Markov. De esta forma solo nos resta definir cómo elegir las futuras palabras en base a la anterior.

Para ello, vamos a definir a la probabilidad de que la palabra B aparezca despues de la palabra A como la cantidad de veces que la palabra B apareció justo después de la palabra A, dividida por la cantidad de palabras que aparecieron despues de A en total. Este análisis debe realizarse únicamente en base a los datos guardados en el archivo de datos recibido.

Una vez hecho este análisis, si queremos generar texto para continuar una palabra en particular (llamémosla X), solo tenemos que elegir al azar una palabra de la lista de palabras que tienen alguna probabilidad de suceder a X, ponderando según las probabilidades correspondientes.

Ejemplo:

Supongamos que tenemos los siguientes cuatro mensajes del mismo contacto:

Aquí todos lloran.
Aquí todos rien.
Aquí todos cantan.
Aquí viven todos felices.

Una vez leídos los mismos, procedemos a modelarlo como Cadenas de Markov. Para ello guardamos todas las palabras que aparecen y, para cada una de ellas, guardamos todas las palabras que la suceden así como su cantidad de apariciones.

Para este texto en particular, tendremos guardada la siguiente información:

Comienzo de oración: (Aquí - 4 apariciones)
'Aquí': ('todos' - 3 apariciones, 'viven.' - 1 aparición)
'todos': ('lloran.' - 1 aparición, 'rien.' - 1 aparición, 'cantan.' - 1 aparicion, 'felices' - 1 aparicion)
'lloran.': (fin de oración - 1 aparicion)
'rien.': (fin de oración - 1 aparicion)
'cantan.': (fin de oración - 1 aparicion)
'viven': ('todos' - 1 aparicion)

Con esta información, la generación no es más que elegir una palabra de comienzo e ir "recorriendo" las relaciones.

Primero buscamos qué palabras son las que comenzaron oraciones en nuestro texto original:

Comienzo de oración:
    Aquí: 4 apariciones
(4 palabras totales)

Sólamente la palabra Aquí puede ser usada para crear una nueva oración de forma pseudo-aleatoria.

Notar también que podríamos usar directamente la fórmula propuesta anteriormente, siendo A el comienzo de oración y siendo B la palabra Aquí. Como B aparece 4 veces luego de A y la cantidad total de palabras que aparecen luego de A es 4, entonces la probabilidad de que Aquí sea seleccionada es 1, lo cual significa que siempre será elegida.

El texto generado hasta ahora es:

Aquí

Ahora vemos las palabras que pueden aparecer luego de Aquí:

Aquí:
    todos: 3 apariciones
    viven.: 1 aparición
(4 palabras totales)

Aplicando nuevamente la fórmula podemos observar que hay 3/4 de probabilidad de que a Aquí le siga todos y 1/4 de probabilidad de que le siga viven. En este caso cualquiera de las dos puede ser seleccionada, pero, de generar una cantidad de textos mas grande, es más probable que la palabra todos sea seleccionada por encima de la palabra viven.

IMPORTANTE: En Python existe el módulo random que incluye utilidades relacionadas a la generación de números aleatorios. Se recomienda investigar la función random.choice() teniendo en cuenta que la misma hace una elección equiprobable, donde por ejemplo random.choice(['Aquí', 'todos']) producirá Aquí el 50% de las veces y todos el otro 50%. Un problema a resolver es lograr que la elección sea pesada, es decir que Aquí tenga 75% (3/4) de probabilidad de ser elegida.

Se permite el uso de las otras funciones del módulo.

Para continuar con el ejemplo, vamos a suponer que salió viven (a pesar de ser la palabra menos probable). Entonces el texto quedaría como:

Aquí viven

Si ahora vemos el contexto (las palabras que están despues) de la palabra viven:

viven:
    todos: 1 aparición
(1 palabra en total)

La única palabra que puede salir es todos, por lo que el texto va quedando como:

Aquí viven todos

Vemos el contexto de la palabra 'todos':

todos:
    lloran.: 1 aparición
    rien.: 1 aparición
    cantan: 1 aparición
    felices: 1 aparición
(4 palabras totales)

En este caso vemos que todas las palabras siguientes tienen la misma probablidadad de aparecer. Supongamos que para el ejemplo sale la palabra lloran..

Si vemos el contexto de lloran.:

lloran:
    fin de texto: 1 aparición
(1 palabra en total)

Notamos que únicamente está el 'fin de texto', entonces terminó nuestro texto.

Finalmente, nuestro nuevo texto quedó así:

Aquí viven todos lloran.

Consideraciones sobre la generación de mensajes

  • Los mensajes a generar no necesariamente tendrán sentido. Estamos aplicando una técnica muy simple de generación de texto que solo tiene en cuenta el patrón de uso de palabras, y no el contexto o significado de las mismas.
  • Los mensajes generados serán más variados e interesantes si el chat a analizar tiene muchos mensajes. Si se están usando los archivos ofrecidos por el enunciado, se recomienda tomar el chat de Friends que tiene la mayor cantidad de mensajes.
  • Al calcular las probabilidades, se pueden saltear aquellos mensajes que tengan muy pocas palabras. Al ignorar los mensajes simples de respuesta como "Si." y "Jajaj", logramos conseguir probabilidades de uso de palabras variadas y que formen oraciones completas.
  • En nuestra implementación no será necesario considerar inicio de texto. Para comenzar los mensajes podemos elegir cualquiera de todas las palabras y de ahí empezar a seguir la "cadena" como se mostró en el ejemplo.
  • Terminarán los mensajes al llegar a una cantidad de palabras fija o encontrar una palabra a la que no le siguen palabras.
    • Otra manera de terminar con el mensaje podría ser si se considera la finalización de un mensaje como otra palabra con probabilidad de ocurrir. Si llegamos a dicha "palabra", finalizamos el mensaje.

Restricciones y condiciones de entrega

  • El programa nunca debe finalizar con una excepción. En el caso de un error, se debe mostrar por pantalla al usuario el problema que ocurrió.
  • No se permite el uso de ninguna biblioteca externa.
  • El código debe estar correctamente modularizado, documentado, y seguir las convenciones y buenas prácticas establecidas por el lenguaje y el curso.
  • El código debe ser eficiente, con uso responsable de archivos y eligiendo las estructuras correctas para la solución de los problemas correspondientes.
  • El uso de módulos de la biblioteca estándar de Python deberá ser previamente autorizado por un docente. Los siguientes módulos de la biblioteca estándar están permitidos sin previa autorización: random y csv.
  • No se permite el uso de variables globales (sí el de constantes!).
  • No se permite el uso de las instrucciones eval, exec.
  • No se permite el uso de funcion(*argumentos).

Entrega

  • Se deberán entregar únicamente los archivos .py que se utilicen.
  • No subir los archivos de chats utilizados.
  • La entrega se hará utilizando el formulario de entregas bajo la sección TP2.
